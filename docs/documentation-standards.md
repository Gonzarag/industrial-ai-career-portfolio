# Documentation Standards

## General Principles

Every project should be understandable to both technical and non-technical readers.

A good project should explain:

- What problem is being solved
- Why the problem matters
- What data was used
- What method was applied
- What result was achieved
- What the result means from an engineering perspective
- What the limitations are

---

## README Requirements

Each project README must include:

1. Project summary
2. Engineering problem
3. Objectives
4. Data description
5. Methodology
6. Tools used
7. Screenshots
8. Results
9. Engineering interpretation
10. Limitations
11. Future improvements
